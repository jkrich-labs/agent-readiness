---
signal_name: Dependencies Pinned
---

## Criterion-Specific Fix Guidance

- **Node.js**: Ensure `package-lock.json` (npm), `yarn.lock` (yarn), or `pnpm-lock.yaml` (pnpm) is committed to version control. Run `npm install`, `yarn install`, or `pnpm install` to generate the lockfile if missing. Never add the lockfile to `.gitignore`.
- **Python (pip)**: Generate `requirements.txt` with pinned versions using `pip freeze > requirements.txt`. Better yet, use a modern tool like `uv` (generates `uv.lock`) or `poetry` (generates `poetry.lock`).
- **Python (Poetry)**: Run `poetry lock` to generate `poetry.lock` and commit it.
- **Python (uv/pip-tools)**: Run `uv lock` to generate `uv.lock`, or `pip-compile requirements.in -o requirements.txt` for pip-tools.
- **Go**: `go.sum` is the lockfile equivalent. Run `go mod tidy` and commit both `go.mod` and `go.sum`.
- **Rust**: `Cargo.lock` is generated by `cargo build` or `cargo generate-lockfile`. Commit it for applications (recommended by Cargo docs); for libraries it is optional.
- **Remove lockfiles from `.gitignore`**: Check `.gitignore` for entries like `package-lock.json`, `yarn.lock`, `poetry.lock`, etc., and remove them.
- **CI enforcement**: Use `npm ci` (not `npm install`) in CI to install from lockfile only. For Python, use `uv sync --locked` or `poetry install --no-update`.

## Criterion-Specific Exploration Steps

- Check for lockfile presence: `package-lock.json`, `yarn.lock`, `pnpm-lock.yaml`, `poetry.lock`, `uv.lock`, `Pipfile.lock`, `go.sum`, `Cargo.lock`
- Check `.gitignore` to see if any lockfiles are explicitly ignored
- Identify the package manager in use from `package.json` (packageManager field), `pyproject.toml` (build-system), or presence of config files

## Criterion-Specific Verification Steps

- Confirm at least one lockfile exists and is tracked by git: `git ls-files | grep -E 'lock|go\.sum'`
- Verify the lockfile is not listed in `.gitignore`
- Run the appropriate install command with lockfile-only mode (`npm ci`, `uv sync --locked`) and confirm it succeeds
